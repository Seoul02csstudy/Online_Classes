##### 1. 3 way handshake란?

- TCP의 클라이언트-서버의 통신 방법중 하나
- 클라이언트-> 요청(syn) -> 서버
- 서버 -> 응답(ack+syn) -> 클라이언트
- 클라이언트 -> 재응답(ack) -> 서버

##### 2. HTTP와 HTTPS란?

- HTTP: 클라이언트와 서버가 자원(텍스트)을 주고받을때 쓰는 통신 규약

- HTTPS: 자원(텍스트)를 암호화한다.

##### 3. GET, POST Method

- GET: http 메시지 header부분에 담겨져 전송, 보안에 취약
- POST: http 메시지 body부분에 담겨져 전송, 보안에 상대적 강함
